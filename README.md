## 📌 Sobre o Projeto
Este repositório contém os artefatos do MVP da Sprint de "Projeto da Interação Humano Computador e da Interface do Usuário". O foco deste trabalho é o **MedRoom ADM**, um sistema administrativo desenvolvido para gerenciar produtos, clientes e contratos da MedRoom, uma empresa de tecnologia educacional voltada para a área da saúde.

Neste reprojeto, o escopo foi direcionado ao **fluxo de compra e gerenciamento de assinaturas**, buscando tornar o processo mais simples, transparente e seguro para médicos residentes que possuem uma rotina intensa.

## 🎯 Objetivos do Reprojeto
Foram definidos dois objetivos principais para a interação:
1. **Adquirir um produto:** Permitir que o usuário adquira um produto da MedRoom por meio da contratação de uma assinatura, escolhendo o plano desejado e concluindo o pagamento.
2. **Atualizar forma de pagamento:** Permitir que o usuário altere uma assinatura existente, atualizando o plano contratado e a forma de pagamento para garantir a continuidade do acesso aos conteúdos.

## 🧑‍⚕️ Personas e Cenários
O design foi guiado pelas necessidades de duas personas principais:
* **Pâmela (Médica Residente):** Busca contratar uma plataforma digital para auxiliar nos estudos, mas precisa de clareza nos planos e benefícios.
* **Rafael (Médico Residente):** Precisa atualizar os dados de pagamento de forma rápida e intuitiva após perder seu cartão de crédito, evitando a interrupção de seus acessos.

## 🛠️ Artefatos Desenvolvidos
* **Diagramas MoLIC:** Modelagem da interação para os dois fluxos principais (disponíveis no relatório em PDF).
* **Wireframes / Protótipo de Alta Fidelidade (UI):** Interfaces desenhadas para os cenários de login, catálogo de produtos, checkout, gerenciamento de assinaturas e troca de plano/pagamento.
